# Contributing Guidelines

Thanks for helping improve **Awesome Invasive BCI Decoding**! This list aims to be a high-signal, well-organized reference for the invasive brain–computer interface decoding community.

## Scope

This list covers **invasive** neural decoding only:

- ✅ Intracortical microelectrode arrays (Utah array, microwire)
- ✅ ECoG (electrocorticography)
- ✅ Stereo-EEG / depth electrodes
- ✅ Neuropixels / high-density silicon probes
- ❌ EEG / MEG / fNIRS / fMRI and other non-invasive modalities (a companion list is planned — see the README "coming soon" section)

If your contribution is non-invasive, it does not belong here yet.

## How to Add an Entry

1. **Pick the right axis.** The list is organized along three axes — *acquisition modality*, *task*, and *research direction*. Add your paper under the single best-fit heading. Cross-list a paper under a second axis **only** if it is a genuine landmark that readers would expect to find in both places.

2. **Use the standard format:**

   ```
   - <emoji> **Title** (First-author / Lab, Year, Venue) — one-line contribution. [paper](url) · [💻 code](url) · [📊 data](url)
   ```

   Emoji legend: 📄 paper · 💻 code · 📊 dataset · 🏆 benchmark · 📚 survey

3. **One line of substance.** The contribution note should say *what is new / why it matters* in a single clause — not just restate the title.

4. **Verify every link.** Every URL must resolve. Prefer, in order: official publisher page / DOI → lab page → arXiv / bioRxiv → aggregator (Semantic Scholar, ResearchGate). Avoid paywalled-only links when a free version exists.

5. **No future-dated / unverifiable preprints.** If a bioRxiv/arXiv link does not actually resolve to a real paper, leave it out.

6. **Keep ordering sensible.** Within a subsection, roughly chronological or grouped by sub-theme is fine; match the surrounding style.

## Submitting

- Open a **pull request** with your additions, or
- Open an **issue** if you want to discuss placement / suggest a paper for someone else to add.

PRs that add a batch of papers in one area, fix dead links, or improve categorization are all welcome.

## Quality Bar

Prefer peer-reviewed work and influential preprints from established venues (Nature/Science family, NEJM, Cell, eLife, NeurIPS/ICML/ICLR, J. Neural Eng., etc.). Niche or very early preprints are fine if they are clearly relevant and the link resolves.

## Code of Conduct

Be respectful and constructive. This is a community resource — assume good faith.
